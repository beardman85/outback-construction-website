## Rules Claude Code Must Follow

1. **Zero client-side JS** unless a feature truly requires it. No React islands for static content.
2. **Every page** gets: unique `<title>` (under 60 chars), unique meta description (under 155 chars), canonical URL, OpenGraph tags, and exactly one `<h1>`.
3. **NAP consistency:** business name, address, and phone are rendered ONLY from `src/data/business.json`. Never hardcode them.
4. **Images:** use Astro's `<Image />` component вЂ” AVIF/WebP, explicit width/height (prevents CLS), lazy-load below the fold, eager + `fetchpriority="high"` for the hero.
5. **Fonts:** self-hosted, `font-display: swap`, preloaded in Base.astro. Max 2 families.
6. **Phone numbers** are always `<a href="tel:...">` links with a GTM click event.
7. **Sitemap** via `@astrojs/sitemap`; `robots.txt` points to it.
8. Lighthouse budget: **95+ on all four categories, mobile.** If a change drops below that, fix it before committing.

## Page Templates

**Service page (`/services/[service]`):** H1 = "{Service} in {Primary City}" в†’ trust bar (rating, years, licensed/insured) в†’ problem/solution copy (400вЂ“800 words, genuinely useful) в†’ process steps в†’ pricing guidance в†’ FAQ (5вЂ“8 questions) в†’ reviews relevant to the service в†’ CTABlock.

**Location page (`/locations/[city]`):** must contain genuinely local content вЂ” neighborhoods served, local landmarks/context, city-specific FAQs, reviews from that city. **Never** template-swap only the city name; Google treats that as doorway pages.

**Google Ads landing pages:** headline mirrors the ad group's keyword (message match = Quality Score), phone number above the fold, form with 3 fields max (name, phone, issue), no nav links that leak clicks, testimonials + license numbers for trust. Keep indexable only if content is substantial; otherwise `noindex`.

## Conversion Tracking

- GTM container in Base.astro with **Consent Mode v2** defaults set before the GTM snippet loads.
- GA4 events: `generate_lead` (form success), `phone_call_click` (tel: links), `book_online` if applicable.
- Import these as conversions in Google Ads (or use Ads tags in GTM). Enable **Enhanced Conversions** вЂ” pass hashed email/phone from the form handler.
- Form flow: POST to `/api/lead` в†’ Worker validates + emails/webhooks the lead в†’ returns success в†’ client fires the GTM event в†’ redirect to `/thank-you` (backup conversion trigger).
- Add **CallRail or similar dynamic number insertion** if running multiple ad channels, injected via GTM.

## Cloudflare Pages Deployment

1. Push repo to GitHub.
2. Cloudflare dashboard в†’ Workers & Pages в†’ Create в†’ connect repo.
3. Build command: `npm run build` В· Output dir: `dist` В· Framework preset: Astro.
4. Add custom domain; Cloudflare handles DNS + SSL (or move the domain's DNS to Cloudflare for full speed benefit).
5. Set `_headers` file for caching: long-cache hashed assets, short-cache HTML.
6. Preview deploys are automatic per branch вЂ” use them for client review.
